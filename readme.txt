--ΕΚΤΕΛΕΣΗ ΠΡΟΓΡΑΜΜΑΤΟΣ--
Για την δημιουργία του εκτελέσιμου για το B+ δέντρο, τρέξτε την εντολή:
make bp

Για να τρέξετε το εκτελέσιμο:
make run
|-> Στην make run το μόνο που τροποποιήσαμε είναι ότι
    προσθέσαμε την εντολή
	    rm -f data.db
    για να σβήνει κάθε φορά (εάν υπάρχει) το αρχείο data.db .

Αρχικά, στην συνάρτηση main(), έχουμε αφήσει την δομή της main που μας
είχατε παραδώσει στο zip αρχείο, επομένως με το make run θα σας δείχνει
τη λειτουργικότητα των δικών σας συναρτήσεων και ψάχνει το κλειδί
340, το οποίο βάλαμε εμείς, μιας που είδαμε ότι υπάρχει για τις συγκεκριμένες
εγγραφές της εκτέλεσης αυτής. Οι συγκεκριμένες τιμές στις οποίες αναφερόμαστε
και "πειραξάμε" είναι μέχρι ποιό νούμερο μπορεί να υπάρξει εγγραφή, το οποίο
το βάλαμε ίσο με 1000 στο αρχείο record.c, και το RECORDS_NUM που του
αρχικοποιήσαμε σε 800.

Για να δείτε περισσότερο την λειτουργικότητα του προγράμματός μας
στην συνάρτηση main έχουμε αφήσει ως σχόλιο την εντολή
  our_main();
Η οποία είναι μια συνάρτηση που φτιάξαμε για να δείξουμε περαιτέρω
τη λειτουργικότητα του προγράμματός μας. Αρχικά το μέγιστο νούμερο
που μπορεί να έχει μια εγγραφή το αφήσαμε 1000 στο αρχείο record.c 
και το πρόγραμμά μας τρέχει για 2000 εισαγωγές εγγραφών για να επιδείξει
ότι δεν δέχεται διπλότυπα. Κάνει δύο αναζητήσεις, μια η οποία για τις
συγκεκριμένες τιμάς δεν υπάρχει και μία, αντίστοιχα, που να υπάρχει.
Επιπλέον, η δικιά μας συνάρτηση εκτυπώνει και το bplus δέντρο, για να μας
βοηθήσει εμάς στην υλοποίηση, εάν δεν θέλετε να το εκτυπώνει κάντε 
comment τη γραμμή 125 στο αρχείο bp_main.c .

Για να τρέξετε τη συνάρτησή μας, κάντε την γραμμή της uncomment και κάντε
τις δύο συναρτήσεις σας comment out.

--ΣΧΕΔΙΑΣΤΙΚΕΣ ΕΠΙΛΟΓΕΣ--
Αποφασίσαμε το split στα data nodes και στα index nodes να ακολουθούν
τη λογική του right - heavy, δηλαδή στα data nodes επιλέγουμε να προωθήσουμε
το μεσαίο κλειδί εφ'όσον έχουμε λάβει υπόψιν το νέο κλειδί με indexing που
ξεκινάει από το 0 και στα index nodes επιλέγουμε να προωθήσουμε το μεσαίο
κλειδί και στη περίπτωση που ο αριθμός των κλειδιών είναι περιττός, το 
κλειδί που αντιστοιχεί στο floor της διαίρεσης με το 2.
